## D-6: 期待値ベース買い目選定

### 現状
composite 順位ベース (col1=◎, col2=◎○▲, col3=◎○▲) で機械的にフォーメーション。

### 改善案
1. 三連複 ticket の期待値 = sum(win_prob_a × win_prob_b × win_prob_c × payout) - stake
2. 期待値 > 0 の ticket のみ採用
3. または期待値 TOP-N の ticket のみ採用 (N = 自信度依存)

### 課題
- payout 推定が必要 (estimate_sanrenpuku_odds 等)
- ML 確率の calibration 必要 (Phase 3 で改善済み)
- 期待値計算は組合せ爆発 (max 8 頭で 56 combo)

### 工数
3-5 日

### 着手条件
A-3 完了 + Phase 3 Isotonic 較正の dashboard 経路統合 (USE_POST_CALIBRATOR=True 切替検討)
