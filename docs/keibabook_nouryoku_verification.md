# 競馬ブック「能力表HTML」をネット競馬の代替とする検証

## 目的

ネット競馬（netkeiba）で400エラーが多発するため、競馬ブックの**能力表HTML**を出走表取得の代替として使えるか検証する。

## 対象URL

| 区分 | URL形式 | 例 |
|------|--------|-----|
| 地方 | `https://s.keibabook.co.jp/chihou/nouryoku_html/{16桁ID}` | [浦和1R 2026/2/25](https://s.keibabook.co.jp/chihou/nouryoku_html/2026021302010225) |
| 中央 | `https://s.keibabook.co.jp/cyuou/nouryoku_html/{12桁ID}` | [東京11R G1 例](https://s.keibabook.co.jp/cyuou/nouryoku_html/202601040811) |

- 地方の16桁: 既存の `keibabook_training.py` で使っている NAR KB race_id と同じ（nittei の出馬表リンクから取得）。
- 中央の12桁: `jra_netkeiba_to_kb_id(netkeiba_race_id)` で netkeiba race_id から変換可能。

---

## 検証結果（未ログイン）

| 項目 | 結果 |
|------|------|
| **地方 (chihou)** | アクセスすると「**ログインして下さい**」と表示。レース見出し（1R・C3-5組・1400m・賞金）とタブ（出馬表/オッズ/能力表HTML/レース結果等）は一部見えるが、**能力表の実データ（馬名・成績表など）は取得できていない**。 |
| **中央 (cyuou)** | 同様にログイン促し。能力表本文はほぼ返っておらず、ナビのみ。 |
| **有料案内** | 地方ページに「全データをご覧になりたい場合は、競馬ブックSmartにご登録下さい」「有料メニュー登録」の文言あり。**ログイン後も一部は有料の可能性**。 |

**結論: 未ログインでは能力表の実データは取れない。ログイン必須。**

---

## 検証結果（ログイン済み・実機）

検証スクリプト `scripts/fetch_keibabook_nouryoku.py` を実行し、**競馬ブックSmartプレミアム**でログインした状態で能力表HTMLを取得した。

| 対象 | 結果 |
|------|------|
| **地方** (chihou) | 約 **203KB** のHTMLを取得。`use:smartpremium` のキャッシュコメント付き。 |
| **中央** (cyuou) | 約 **22KB** のHTMLを取得。 |

### 地方能力表のHTML構造（確認済み）

- **テーブル**: `table.nouryoku_html_table`、行は `tr.js-umaban01` ～ `js-umaban09` 等で馬番ごと。
- **列**:  
  - `td.wakban` … 枠番  
  - `td.umaban` … 馬番  
  - 馬名 … `span.kbamei` および `a[href^="/db/uma/"]`（馬名＋DB用ID相当のパス）  
  - 騎手 … `span.kisyu`、リンク `a[href^="/db/kisyu/"]`  
  - 斤量 … `span.kinryo`  
  - 前走 … `th.zensou`（前走・2走前・3走前）に対応するセル。説明文に「距離・コース・タイム・騎手・斤量・ペース・前半3F・上がり3F・通過順位」等の記載あり。
- 出走表に必要な **枠番・馬番・馬名・騎手・斤量** は揃っている。**前走サマリも列として存在**し、パースすれば過去走の要約は取得可能。

**結論: ログイン済みであれば、能力表HTMLを出走表の代替として利用できる。**  
（馬の netkeiba 馬ID との対応は、KB の `/db/uma/XXXXXX` 等のIDを利用するか、馬名・騎手で突き合わせる必要あり。）

## ログイン後の利用可否（方針）

- 本プロジェクトでは既に **競馬ブックの認証** を実装済み（`KeibabookClient`、調教・厩舎の話取得で使用）。
- 認証情報: `KEIBABOOK_ID` / `KEIBABOOK_PASS` または `~/.keiba_credentials.json` の `keibabook_id` / `keibabook_pass`。
- **同じセッションで能力表HTMLのURLにGETすれば、ログイン済みならHTMLが返る想定**。
- 次のステップ:
  1. **ログイン済みで能力表HTMLを1件取得**し、実HTMLを保存して構造（テーブル・クラス・馬番・馬名・過去走の有無）を確認する。
  2. 能力表に「出走表に必要な項目」（馬番・馬名・枠順・騎手・斤量・過去走の要約など）が含まれているか確認する。
  3. netkeiba の `RaceInfo` / `Horse` / `PastRun` に**対応するデータが揃うか**を確認し、揃うなら能力表パーサーを実装して「netkeiba 失敗時のみ KB 能力表にフォールバック」する案が現実的。

## netkeiba とのデータ対応

現在 netkeiba から取得している主な項目:

| 用途 | netkeiba | 競馬ブック能力表で揃うか |
|------|----------|---------------------------|
| レース一覧（日付→race_id） | 日別ページ | KBは「nittei」（日程）でレース一覧あり。既に NAR 用に nittei を参照済み。 |
| 出走表（馬番・馬名・枠・騎手・斤量・オッズ等） | shutuba.html / newspaper.html | 能力表HTMLに出走表に相当する表があるか要確認。 |
| 馬の過去走（走破タイム・着順・ペース等） | 馬詳細ページ | 能力表は「能力値」中心のため、**過去走の生データが同じ粒度で揃うか要確認**。 |
| オッズ | odds ページ | KB には「オッズ」タブが別URL。必要なら別取得。 |

→ **出走表の代替としては有望。過去走は能力表の表の内容次第で、不足分は netkeiba 馬詳細にだけ頼るか、別ソースを検討する必要あり。**

## 競馬ブックを採用する理由（補足）

- **過去3走だけ見るなら競馬ブックで十分**  
  分析で「直近3走」だけを参照する前提なら、能力表の前走・2走前・3走前の列で賄える。
- **地方競馬で「最初の3F」タイムが出ているのは競馬ブックだけ**  
  netkeiba 等では地方の前半3F（最初の3F）が揃わないケースがあり、**ペース・展開分析では競馬ブックが唯一のソース**になり得る。パーサー実装時に「最初の3F」の抽出を優先すると価値が高い。
- **見にくい部分**  
  HTML・表のレイアウトには見づらい箇所があるが、データとしては利用可能。パース時はクラス名（`zensou`・ペース関連のクラス等）を頼りにするとよい。

## 推奨アクション

1. **検証スクリプトの実行**  
   `scripts/fetch_keibabook_nouryoku.py` で、ログイン済みセッションを使って能力表HTMLを1件取得し、`data/cache/keibabook/` などに保存する。
2. **保存したHTMLの目視確認**  
   テーブル構造・馬名・枠順・騎手・斤量・過去走の有無を確認する。
3. **方針決定**  
   - 必要な項目が揃う → 能力表用パーサーを実装し、netkeiba 失敗時にフォールバック。  
   - 過去走が不足 → 出走表のみ KB、過去走は netkeiba または別APIに限定する。
