# P3 新規データ取得可能性 予備調査 (2026-06-26)

> roadmap P3「市場が価格に織り込んでいない新規データ」(含水率/クッション値/風) の取得可能性・リーク無・パイロット設計。本番非改変の予備調査。実スクレイピング着手は master 承認後。

## 1. 背景 (なぜ P3 が唯一の出口か)
指数・印・composite いじりの天井を**四方向から確認済**:
- P1-a odds残差化 = 無効確定 (市場効率の壁)
- P1-b 穴馬ダンピング撤廃 = 無効
- P0-γ 物差し (engine印 vs prob印) = 同一レースで印質ほぼ互角 (+1.2pt)
- レース選択効果 = engine印は「荒れ回避の守りの選択眼」だが見送った先も赤字 (黒字化の鍵でない)

→ 既存データの再加工では控除率を破れない。**市場が価格に織り込んでいない新規データ** = 唯一の前進方向 ([[project_improvement_roadmap]] P3)。

## 2. 現状のデータ粒度 (調査結果)
- **馬場状態**: `src/scraper/netkeiba.py:1036-1059` が「良/稍重/重/不良」をパース → `src/ml/features.py:30-31 CONDITION_MAP` で 4段階数値化 (良0/稍1/重2/不良3) → FEATURE_COLUMNS `condition`。
- **含水率%・クッション値・風 = 未取得** (ばんえいの水分量 `netkeiba.py:1872` のみ例外)。
- = 市場(オッズ)も「良/重」の粗い4段階は知っている。**含水率%の細かい連続値・クッション値・風の数値は一般購入者が見ない = 市場が完全には織り込んでいない可能性**。

## 3. 取得可能性・リーク無の評価 (JRA クッション値/含水率)
| 項目 | 評価 | 根拠 |
|---|---|---|
| 過去データ | ✅ 公開 | JRA公式 archive (jra.go.jp/keiba/baba/archive/・2020年〜) + netkeiba まとめ。学習データになる |
| 予測時点確定(リーク無) | ✅ リーク無 | 含水率=開催**前日(金)昼過ぎ**+当日9:30 / クッション値も前日・当日。**前日18時予測にも当日17時finalizeにも間に合う** |
| 取得方法 | ⚠️ 新規scraper要 | netkeiba本体(`netkeiba.py:1142-1160`)に無し。JRA公式 archive (HTML/PDF) の新規パーサが必要 |
| **JRA限定** | ⚠️ **制約** | クッション値は**JRAのみ**。NAR含水率は一部場のみ。本システムはNAR主力(14会場) → 効果がJRAサブセットに限定 |
| 風 | ⚠️ 別ソース | JRA「天候」は晴/曇/雨の粗粒度。風速/風向は気象庁データ(別スクレイパー) |

## 4. パイロット設計 (master 承認後に実装)
1. **取得**: JRA公式 archive (jra.go.jp/keiba/baba/) から 2024-2026 のクッション値・含水率(芝/ダ・地点別%)をスクレイピング → 日付×競馬場で race_id 紐付け。★netkeiba非依存・JRA公式のみ・レート制限遵守。
2. **特徴量化**: 現状 `condition` 4段階に加え `cushion_value`(連続)・`moisture_goal`/`moisture_corner`(%)・`wind_speed`/`wind_dir` を追加。`features.py` + `lgbm_model.py FEATURE_COLUMNS` 同時追加 ([[feedback_feature_cols_sync]])。
3. **パイロット測定**: **JRAレース限定**で 2026-01 WF を baseline vs +新特徴量で比較 → ΔROI/ΔAUC。SHAP で新特徴量の寄与確認 (popularity/odds が gain0 だった前例 → 新データが本当に効くか検証)。
4. **合格ライン**: ΔROI が有意 (+数pt) かつ SHAP寄与が非ゼロなら本実装へ。NARへの拡張可能性(NAR含水率)を別途調査。

## 4.5 PDFパース実証 ✅ (2026-06-26 完了)
`scripts/parse_jra_baba_pdf.py`(プロトタイプ・pdfplumber)で **JRA archive PDF のパース成功**を実証。中山2026年1回PDF → 13測定日を構造化:

| 取得項目 | 例(中山 1/3 土) | 備考 |
|---|---|---|
| 芝クッション値 | 9.1 | **芝のみ**(JRA仕様・ダートはクッション値なし) |
| 芝含水率(ゴール前/4角) | 12.5 / 13.2 | % |
| ダート含水率(ゴール前/4角) | 7.2 / 8.2 | % |
| 芝使用コース | B | A/B/C..(内柵位置) |

- **データ粒度**: 開催日次(1日1測定・朝)。レース毎の時間変動は無し → 日次の馬場硬さ/含水率を全レース共通の特徴量に。
- **リーク無 再確証**: 金曜(開催前日)の測定値もPDFに含まれる(1/9・1/16・1/23 金) → 前日予測でも前日値が使える。当日予測は当日朝値。**予測時点で利用可能な最新値を使えばリーク無**。
- **依存**: `pdfplumber`(導入済・Pillow/pdfminer.six/pypdfium2)。

## 5. リスク・留保
- **JRA限定が最大の制約**: 本システムの主力はNAR(14会場)。クッション値JRA限定では全体ROIへの寄与が薄い恐れ。NAR含水率(一部場)の可用性を要追加調査。
- **新データが市場に既に織り込まれている可能性**: クッション値は2020年以降 netkeiba まとめ等で公開済 → 一部の市場参加者は利用。「市場が織り込んでいない」前提の再検証が必要 (SHAP/ΔROIで実証)。
- **過去データの粒度・欠損**: archive の地点別%・PDFパース精度・2020年以前の不在。
- 実スクレイピングは netkeiba事故の教訓 ([[feedback_netkeiba_concurrent_throttle]]) 同様、JRA公式もレート制限・逐次取得を厳守。
