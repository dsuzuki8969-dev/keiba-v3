# さらなる精度向上のための最高案

実装済みの改善を超えて、劇的に精度を高める追加アイデアを整理した。

---

## 凡例

- **期待効果**: 実装により得られる精度向上の度合い
- **難易度**: 実装・データ取得の負荷
- **根拠**: 競馬理論・先行研究・データ上の裏付け

---

## 最優先レベル

### 1. **血統×距離適性の統合**

| 項目 | 内容 |
|------|------|
| **現状** | 血統情報（父・母）は未使用。距離適性は過去走のコース別成績のみ。 |
| **根拠** | 父馬の産駒は「芝マイル」「ダート長距離」など傾向が明確。同じ過去走成績でも父系が異なれば適性が違う。 |
| **期待効果** | コース適性偏差値・能力評価の補正に効く。特に未勝利・新馬の予測精度が上がる。 |
| **難易度** | 中。netkeiba に父馬・母父の成績データがある。血統×距離別勝率を事前に集計するDBが必要。 |
| **取得** | db.netkeiba.com の馬ページに父馬リンク。父馬の産駒成績（距離別）をスクレイプして蓄積。 |

---

### 2. **馬体重・増減の定量評価**

| 項目 | 内容 |
|------|------|
| **現状** | PastRun に horse_weight, weight_change があるが、能力・展開への反映は限定的。 |
| **根拠** | 「馬体重が増えた=仕上がり」「大幅減=調子落ち」は定説。±10kg 以上の増減で複勝率が大きく変わる。 |
| **期待効果** | 能力偏差値の補正、勝負気配の精度向上。 |
| **難易度** | 低。既に horse_weight は取得済み。馬体重×増減のヒストグラムから補正係数を算出すればよい。 |
| **取得** | レース結果・馬の過去走に「馬体重(増減)」がある。course_db に含めれば集計可能。 |

---

### 3. **騎手×馬のコンビ成績**

| 項目 | 内容 |
|------|------|
| **現状** | テン乗り（初コンビ）は -1pt のペナルティのみ。過去コンビの勝率・複勝率は未使用。 |
| **根拠** | 「この騎手が乗ると勝率が高い」馬が存在。コンビ成績 10 走以上で 30% 超なら明確な相性。 |
| **期待効果** | 騎手展開影響（H-3）がデータ駆動に。乗り替わり時の評価精度向上。 |
| **難易度** | 中。馬の過去走に jockey_id があるので、馬×騎手の組み合わせで集計可能。 |
| **取得** | course_db の PastRun に jockey_id がある。馬の過去走から馬×騎手別成績を蓄積。 |

---

## 高優先レベル

### 4. **クラス・格の落差補正**

| 項目 | 内容 |
|------|------|
| **現状** | 基準タイムはコース単位で均一。G1 出走馬と未勝利馬の成績を同列に扱う。 |
| **根拠** | 格上挑戦時は実力以上の結果になりやすい。格下だと油断や手抜きの可能性。 |
| **期待効果** | 能力偏差値の補正。格落ち・格上時の解釈が正確に。 |
| **難易度** | 低。PastRun.grade がある。grade 同士の対戦結果を集計して補正テーブルを構築。 |

---

### 5. **休み明けの精密判定**

| 項目 | 内容 |
|------|------|
| **現状** | detect_long_break で長期休養を検出。休み明け回収率（recovery_break）は厩舎単位。 |
| **根拠** | 休み日数（30 日 / 60 日 / 90 日以上）で適性が異なる。調教師ごとの「休み明け1走目」勝率もある。 |
| **期待効果** | 勝負気配・穴馬検知の精度向上。 |
| **難易度** | 低。race_date と直近走の race_date の差で休み日数を算出。調教師×休み日数帯で集計。 |

---

### 6. **オッズ・人気との整合性スコア**

| 項目 | 内容 |
|------|------|
| **現状** | 総合偏差値とオッズは別々に算出。偏差値が高くオッズも低い「本命」と、偏差値は高いがオッズが割れている「穴」の区別が曖昧。 |
| **根拠** | 市場（オッズ）とモデルの乖離 = 期待値の源泉。乖離が大きい馬ほど買い目候補。 |
| **期待効果** | 買い目生成・期待値算出の精度向上。 |
| **難易度** | 低。既に odds, composite がある。composite から算出した「モデル勝率」とオッズから逆算した「市場勝率」の差をスコア化。 |

---

## 中優先レベル

### 7. **牝馬の繁殖・発情周期**

| 項目 | 内容 |
|------|------|
| **根拠** | 発情馬はパフォーマンスが落ちる場合がある。繁殖入り直前のラストランは本気度が高い。 |
| **難易度** | 高。発情の有無は公表データにない。牝馬の出走間隔や成績パターンから推定する必要。 |
| **取得** | 現状の無料データソースでは難しい。JRA-VAN 等に情報があるか要調査。 |

---

### 8. **コース×開催×日付の時系列バイアス**

| 項目 | 内容 |
|------|------|
| **根拠** | 同一コースでも、開催初日と最終日で馬場が変わる。季節（札幌夏・中山冬）で傾向が異なる。 |
| **期待効果** | 基準タイム・ペース予測の補正。 |
| **難易度** | 中。course_id に venue が含まれる。race_date から開催・曜日を抽出し、開催内での経過日数で補正。 |

---

### 9. **末脚の安定性（バラつき指標）**

| 項目 | 内容 |
|------|------|
| **現状** | 末脚タイプ（爆発末脚等）は平均的な傾向。同一馬の上がり3Fの標準偏差は未使用。 |
| **根拠** | 安定した末脚の馬はスローペースでも信頼できる。バラつきが大きい馬はペース依存が強い。 |
| **期待効果** | 展開偏差値の補正。ペース×末脚の相性判定が精密に。 |
| **難易度** | 低。馬の過去走の last_3f_sec の標準偏差を算出するだけ。 |

---

### 10. **距離適性の連続値化**

| 項目 | 内容 |
|------|------|
| **現状** | コース適性は course_id（距離含む）別。1600m と 1800m は別コースとして扱う。 |
| **根拠** | 「マイルが得意」な馬は 1600–1800m で一貫して好走。距離を連続値として、最適距離との差で補正できる。 |
| **期待効果** | 初挑戦距離への予測精度。 |
| **難易度** | 中。過去走の distance と成績から、馬ごとの「最適距離」「得意レンジ」を推定。 |

---

## 実装済み（本セッション）

| 項目 | 内容 |
|------|------|
| 通過タイムの代わり | POSITION_SEC_PER_RANK をコース別に course_db から較正。実データで位置取り秒差を精密化。 |
| trainer_baseline_db 永続化 | 分析毎にファイルにマージ保存。蓄積で調教強度判定の σ が機能。 |
| レース単位ラップ拡張 | ペース(H/M/S) を取得し PastRun.pace に付与。pace_last3f_db のサンプル充実に寄与。 |
