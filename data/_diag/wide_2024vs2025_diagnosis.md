**パラドックス**: 2025年は全体として安いワイドが増えているのに WF avg_hit が高い。

**結論**: 市場変化は副次的要因の可能性あり。主要因ではない。

## 4. 仮説 C: results.json 払戻データ仕様変化/品質バグ

### 検証結果: **確定（最高確信度）**

#### 4-1. データソースの変化

| 年 | source | ワイド形式 |
|---|---|---|
| 2024年 | `keibabook_backfill` | 3通り別々の払戻 (250/270/740等) |
| 2025年 | `race_results_db` | 3通りすべて同額 (640/640/640等) |

- 2024-09: 全通り同額レース = 2/1444 (0.1%)
- 2025-03: 全通り同額レース = 1407/1411 (99.7%)
- 2025-09: 全通り同額レース = 1458/1459 (99.9%)

#### 4-2. 具体的なバグ内容

**正しいワイド払戻 (キャッシュHTMLから直接取得)**:
```
202506020101: 5-12=640円, 11-12=4370円, 5-11=8650円  (3通り別々)
202506020102: 9-15=270円, 9-12=350円, 12-15=240円   (3通り別々)
202506020103: 8-16=720円, 4-16=370円, 4-8=400円      (3通り別々)
```

**results.json に記録されたバグデータ**:
```
202506020101: 5-12=640円, 11-12=640円, 5-11=640円  (全部最初の値)
202506020102: 9-15=270円, 9-12=270円, 12-15=270円  (全部最初の値)
202506020103: 8-16=720円, 4-16=720円, 4-8=720円    (全部最初の値)
```

#### 4-3. バグ発生メカニズム

`rebuild_results_from_db.py` が DB の `payouts_json = {}` (空) から
2025年の results.json を生成した際、別の処理が:
1. キャッシュHTMLからワイドを取得
2. `payout_vals[j]` で3通りのコンボにそれぞれの払戻を対応付けるはずが
3. 何らかの実装で `payout_vals[0]` のみを全コンボに適用
4. 結果: 「最初の払戻のみ3通りに複製」

**2025年の「最初の払戻」の分布**:
- 最安値に相当: 51.1%
- 中間値に相当: 25.5%
- 最高値に相当: 23.4%

#### 4-4. ROI 過大評価の定量確認

```
results.json 平均: 975円/3通り
キャッシュ正値 平均: 1277円/3通り
比率: 0.760 (正しい値の76%しか記録されていない)
```

しかし **WF avg_hit (payout_sum / hit) には別の影響**:

- 2024年: 正しい3通り別々の払戻 → WF が ◎-○ combo でヒットした場合の払戻は
  「その combo 固有の払戻」= 主に低額帯 (popularity=1 相当の 200円台)
- 2025年: 3通り全部同額「最初の払戻」→ WF がどの combo でヒットしても同額
  この「最初の払戻」の平均が 2025年で高め (330-370円)

ROI 計算の確認:
```
ROI = payout_sum / (played × 100) × 100
    = hit × avg_hit_payout / (played × 100) × 100

2024-09: 602 × 200.8 / (1423 × 100) × 100 = 84.9% ✓
2025-03: 546 × 331.6 / (1406 × 100) × 100 = 128.8% ✓
```

**結論**: avg_hit の差 (200円 vs 332円) がそのまま ROI 差 (+50pt) の直接原因。  
2025年の avg_hit が高い理由は「全通り同額バグで最初の払戻が適用され、  
この値が 2024年の popularity=1 (安い) より高い中間的な値になっているため」。

---

## 5. 結論

### 真因: **仮説 C「results.json の払戻データ品質バグ」が確定**

- 確証レベル: **高**
- 2024年: `keibabook_backfill` ソースで正しい3通り別々の払戻
- 2025年: `race_results_db` ソースで3通りすべて最初の払戻を複製するバグ
- 2025年の WF avg_hit は「バグで生成された中間的な払戻額」
- 実際のワイド払戻で再計算した場合、2025年の ROI は大幅に異なる可能性

### 付随確認

- コンボ形式: 全期間ダッシュ区切り (`4-8` 形式) で一貫 → 仮説 C コンボ問題は否定
- 馬連は別の問題 (2024-09: ダッシュなし `48`, 2024-12: ダッシュあり `6-12`, 2025: ダッシュなし) が既知
